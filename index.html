WikiTrust 2.0

My Role: Frontend extension and page algorithms, backend API and cloud database abstraction layer.

Turned the original WikiTrust algorithms into a functional MVP cross-browser extension in typescript and processing server backend in python. Our system can calculate a metric of accuracy or "consensus" for every word on Wikipedia using only the article's revision history and author cross contributions. We hope WikiTrust will give readers & editors of Wikipedia better insight into the accuracy of statements and protect against content vandalism.

Source Code: Github | Web: Wikitrust 2.0

Tech: #ts #python #google-cloud #fullstack #algorithms

Web Graphics Experiments

My Role: Everything

In my final project for the UCSC computer graphics course, I wanted to explore graphics technologies beyond the traditional polygonal raster pipeline that powers most real-time 3D visualizations today. I settled on an efficient form of ray tracing called ray marching in the browser using WebGL paired with a trackpad gesture based modeling flow. This project was all about learning for me, so the end website is not as polished as I would normaly do, but I think it nicely shows the power of alternative graphics.

Play it: Ray-Marched Gesture Editor | Raster Gesture Editor | Both work in all browsers, but are best in Safari with a trackpad

Tech: #WebGL #GLSL #UX #js

Geothermal Energy Web Infographic

My Role: Frontend layout, animations & code + part of the reasearch & writing

Goal - For the 2020 US Department of Energy Geothermal Design Challenge, me and two other students wanted to tackle the lack of public awareness around geothermal energies' potential and the technologies that are making broad geothermal energy possible.

Process - We decided to make an interactive infographic website from scratch. The whole process, from brainstorming to research to writing to art, design, & development took about two months. Doing the writing for the infographic was especially challenging because I was writing for a general audience so I needed to break down tricky topics into easily understood chunks. After writing and creating the artwork, we assembled the website in HTML & CSS. I used the Greensock animation library plus micromodal.js for it’s accessibility features and some custom interaction code to handle the rotating planet on all input types & responsive css & js to lay out all the elements on different form factors.

Result - Our site got 2nd place in the challenge & we hope it is reaching a broader audience thanks to the @geothermalfact twitter bot that a teammate wrote which tweets geothermal fun facts and directs people to our site.

Visit: geothermal-future.web.app — I hope you enjoy it and learn something too!

Tech: #js #greensock #responsive #accessability

StudyHub 2

My Role: Everything

A web platform I designed and built for York School that allows students to share, organize, and access educational study resources. Version 2 is built on Angular and uses Firebase, Algolia Search, and Google Cloud Functions running Node.js as a backend. StudyHub evolved continuously through many prototypes where I switched to new server stacks, made the web app responsive, mobile-ready and offline capable, and then did a major overhall of Studyhub after two kind Google employees gave me feedback on the UX and login security of the site. It took me nearly two years to complete the first version and I spend another six months reworking the codebase to be much faster and more feature-rich for version 2. Studyhub was my first real scaled web app project and it has formed the foundation of my web development experience ever since.

Explore: Interactive prototype of the full V1 app | live V2 site (Requires york.org login)

Tech: #angular #ts #pwa #algolia #firebase #google-apis
